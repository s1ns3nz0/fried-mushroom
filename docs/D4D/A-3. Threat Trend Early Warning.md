# A-3. Threat Trend Early Warning — 위협 궤적 조기경보

## 왜 (임팩트)

D4D 파이프라인은 무상태(ADR-004)다 — 사이클마다 위협을 **독립적으로** 판정한다. 이 설계는
멱등·재현성에 좋지만, "위협이 시간에 따라 **악화되고 있다**"는 궤적은 어떤 단일 사이클도
보지 못한다. 예: RAC 가 Medium→Serious→High 로 오르거나 킬체인이 초기→중기→후기로
진행하는 흐름은 각 사이클 단독으로는 그저 "현재 상태"일 뿐, **다가오는 위기**로 인식되지 않는다.

`onboard/trend.py`는 파이프라인 위에 얹는 **관찰자**로, 최근 N 사이클 결과의 궤적을 보고
조기경보를 낸다 — 위협이 임계에 도달하기 **전에** 운용자가 알 수 있게.

## 무엇 (동작)

`assess_threat_trend(results, *, window=None, min_persist=2) -> dict`:

궤적 신호(지속 위협의 첫→끝 비교):

| 신호 | 조건 |
|---|---|
| `rac_escalating` | RAC_ORDER 악화 (예: Medium→High) |
| `kill_chain_advancing` | 초기→중기→후기 진행 |
| `confidence_rising` | 지배 위협 확신도 상승 |
| `persistent_threat` | 같은 threat_event ≥ min_persist 사이클 지속 |

**레벨**: `none`(위협 없음) / `watch`(위협 있으나 안정) / `warning`(악화 감지) /
`critical`(악화 + 최신 RAC=High 또는 킬체인 후기).

### 예시

```
seq: T3[Medium,초기,0.60] → T3[Serious,중기,0.78] → T3[High,후기,0.91]
→ level=critical, escalating=True
  signals=[persistent_threat, rac_escalating, kill_chain_advancing, confidence_rising]
  note: "T3 위협 악화 궤적 감지(...) — RAC Medium→High. 조기경보 레벨=critical."
```

## CRITICAL — 파생 읽기전용 (SCC-1)

- advisory 만 산출한다. 어떤 사이클 결과도 변경하지 않고 입력 시퀀스를 변이하지 않는다
  (`advisory_only=True`, 무변이 회귀로 강제).
- 무상태 파이프라인의 per-cycle 결정을 **대체하지 않는다** — 별도 관찰 레이어다(ADR-004 존중).
- `RAC_ORDER`/`KILL_CHAIN_STAGE_ORDER`(읽기전용)만 참조하며 어떤 상수도 쓰거나 바꾸지 않는다.

## 용도

- 운용자 조기경보 HUD/알림 (위기 도달 전 경고).
- 사이클 로그 스트림 위의 트렌드 배지(대시보드).
- debrief 시 위협 전개 타임라인.

테스트: `tests/integration/test_trend.py` (10건 — RAC/킬체인/확신도 궤적·안정/완화·윈도우·
critical 레벨·무변이).
