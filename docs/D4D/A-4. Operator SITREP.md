# A-4. Operator SITREP — 상황판단 리포트

## 왜 (임팩트)
운용자는 사이클마다 쏟아지는 판정을 일일이 해석할 여유가 없다. 필요한 건 **"지금 주목해야
하나?"**에 대한 한 줄 답과 그 근거다. `sitrep`은 결정 근거(explain, "왜 이 결정")와 궤적 조기경보
(trend, "어디로 가는가")를 융합해 **단일 주의수준**과 헤드라인·권고를 만든다 — 운용자 결정지원의
한 화면.

## 무엇
`build_sitrep(results, *, window=None) -> dict`:
- `explain_cycle(latest)` + `assess_threat_trend(window)` 융합.
- **주의수준** ROUTINE < MONITOR < ALERT < ACT — 현재 RAC × 궤적 악화 × 위협 존재 융합:
  - `ACT`: 심각 RAC(High/Serious)+행동 중, 또는 궤적 critical.
  - `ALERT`: 악화 궤적 또는 고위험(심각 RAC).
  - `MONITOR`: 위협 존재하나 안정.
  - `ROUTINE`: 위협 없음.

### 예시
```
seq: T3[Medium,초기] → T3[Serious,중기] → T3[High,후기,RTL]
→ [ACT] T3(근접 소화기) RAC=High → RTL
  권고: "즉시 주목 — RTL 실행 중, 궤적 critical. 운용자 확인 요망."
```

## CRITICAL — 파생 읽기전용 (SCC-1)
advisory 만 산출한다. 결정을 바꾸지 않고 입력을 변이하지 않으며 `RAC_ORDER`(읽기전용)만
참조한다. 주의수준은 참고용 실행지표이지 결정 자체가 아니다.

## 용도
대시보드 상단 주의수준 배지, 운용자 알림, debrief 요약.
