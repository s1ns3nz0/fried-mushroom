# B-2. GCS Briefing Advisor — RAG 학습루프 폐쇄

## 왜 (문제)

`infra/log`의 RAG 코퍼스(라운드 1~3)는 매 사이클 **위협 판정 → 실제 outcome**을 축적한다.
그 정본 목적은 문서상 *"다음 임무 브리핑 시 NLP confidence 참고자료"* (`docs/RAG-corpus.md §1`)
였다. 그러나 축적만 될 뿐 **소비하는 쪽이 없었다** — `nlp_extract`는 지시서 원문의 어휘만으로
confidence를 산정하고, 과거 유사 임무의 실측 이력을 전혀 참고하지 않았다. 즉 학습루프가
`episode → corpus`에서 **끊겨 있었다**.

`briefing_advisor`는 그 마지막 연결부다: `corpus → 브리핑 advisory`.

```
과거 사이클: raw_log → episode → corpus (축적) ──┐
                                                 │  (이 화살표가 부재했음)
신규 임무:  지시서 → nlp_extract(신호) ──────────┴──► briefing_advisor ──► advisory(참고)
```

## 무엇 (동작)

`build_briefing_advisory(signals, mission_context, posture, store, ...)`:

1. NLP 신호 중 위협 신호(`signal_type="threat"`, `threat=T1..T7`)의 T-코드를 추출.
2. 각 T-코드에 대해 `CorpusStore.retrieve(mission_context, posture, threat_event=T, ...)`로
   과거 유사 임무 레코드를 회수(`posture_tolerance`로 근접매칭 가능).
3. threat_event별 **이력 캘리브레이션**을 요약:
   - `sample_size` — 표본 수
   - `avg_past_confidence` — 과거 판정 평균 확신도
   - `outcome_distribution` — 실제 outcome 분포(예: `{threat_confirmed: 6, false_positive: 2}`)
   - `sufficient_data` — 표본 ≥ `ADVISORY_MIN_SAMPLES`(기본 3)
   - `note` — 사람이 읽는 요약

## 출력 스키마

```jsonc
{
  "advisory_only": true,               // 항상 true — 참고 전용 표식
  "mission_context": "정찰",
  "posture": { "watchcon": 3, ... },
  "generated_ts": 1234,
  "advisories": [
    {
      "threat_event": "T3", "threat_desc": "근접 소화기",
      "sample_size": 8, "avg_past_confidence": 0.82,
      "outcome_distribution": { "threat_confirmed": 6, "false_positive": 2 },
      "sufficient_data": true,
      "note": "T3(근접 소화기): 과거 8건, 평균 과거 확신도 0.82, outcome[threat_confirmed×6, false_positive×2]."
    }
  ],
  "threat_events_without_history": ["T5"]   // 신호는 있으나 이력 없음
}
```

## CRITICAL — SCC-1 준수 (MIL-STD-882E, CLAUDE.md)

- **advisory는 병렬 참고지표만**이다. 결정론 판정도, NLP 신호의 `confidence`도 **절대 변경하지
  않는다**. 출력은 `advisory_only=true`를 항상 싣고, 입력 `signals`를 무변이(non-mutating)로 둔다.
  (회귀 테스트 `test_does_not_mutate_input_signals`가 이를 강제.)
- 결정론 로직(RAC 매트릭스·SIGNAL_TO_THREAT 등)과 완전히 분리된 소비자다. `shared.constants`의
  `THREAT_CATALOG`(설명 조회, 읽기전용)만 참조하며 어떤 상수도 쓰지 않는다.
- pending(`narrative_status="pending"`) 레코드는 회수 계약상 항상 제외된다.
