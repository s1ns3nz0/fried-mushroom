# 1. 문제 인식

지형이 복잡하고 외부 위협이 상존하는 환경에서, UAV가 임무를 수행하는 동안 최적의 경로를 산정하는 일이 점점 더 중요해지고 있습니다.

그런데 "최적"이란 무엇일까요? 위협을 피해 살아남기만 하면 최적이라고 할 수 있을까요? 생존성만을 기준으로 경로를 정하면 기체는 안전하게 돌아오겠지만, 정작 임무는 완수하지 못했을 수 있습니다. 반대로 임무 완수만을 기준으로 삼으면, 감수해도 될 위협인지 판단 없이 기체를 잃을 수도 있습니다.

즉 진짜 "최적"은 생존성 하나만으로 정의될 수 없습니다. 그 임무가 무엇을 위한 것인지, 무엇을 지켜야 하고 무엇을 감수할 수 있는지를 함께 고려해야 비로소 정의됩니다.

# 2. 해결 방안

저희는 "최적"을 정의하기 위해 '임무'를 판단기준의 중심에 두었습니다. UAV를 띄우는 것 자체가 목적이 아니라, UAV가 수행하고자 하는 군사적 활동 — 즉 임무 — 에 얼마나 영향을 미치는가가 경로산정의 가장 중요한 기준이라고 판단했습니다.

이 판단기준을 구체화하기 위해 저희는 미 국방부(DoD)가 2025년 Cyber Operational Test and Evaluation Guidebook에서 공식적으로 정의한 MBCRA(Mission-Based Cyber Risk Assessment, 임무기반 위험평가) 방법론에 착안했습니다. MBCRA는 원래 사이버 위협을 대상으로 한 프레임워크지만, "핵심기능 식별 → 위협경로 이해 → 위험 우선순위화"라는 3단계 구조는 물리적·전자전 위협을 포함한 UAV 작전 위험평가에도 그대로 적용할 수 있습니다. 저희는 이 구조를 UAV 임무수행 전반의 위험평가로 확장했습니다.

구체적으로, UAV 작전 중 발생하는 다양한 위협(근접위협, GPS스푸핑, 사이버 하이재킹 등)에 대해 위험평가를 수행하고, 그 위협이 임무에 어떤 영향을 미치는지 — 기체손실인지, 임무중단인지, 단순 경계강화로 충분한지 — 를 기준으로 대응 우선순위를 매깁니다. 같은 위협이라도 어떤 임무 중에 발생했는지에 따라 판단이 달라지고, 이 판단 결과를 바탕으로 최적 경로와 대응방식을 산출합니다.

이로써 1장에서 제기한 질문 — "생존성만 확보되면 최적인가" — 에 대한 저희의 답은 명확합니다. 최적은 생존이 아니라 임무 성공에 대한 기여도로 정의되며, 저희 시스템은 이 기준을 위협모델링·위험평가·자동대응 전 과정에 일관되게 반영합니다.

# 3. 아키텍처

해당 서비스는 지상통제센터에 설치된 AI와 온보드 AI로 구성됩니다. 지상통제센터 AI는 기존 UAV 운용에 필요한 항로 정보, 연동된 C4I 정보, 자산 정보 등을 바탕으로 UAV의 임무를 세분화합니다. 또한 비행 종료 후에는 UAV에 축적된 비행 데이터를 학습하여 다음 작전을 위해 UAV의 AI 모델을 업데이트합니다. UAV의 AI는 여러 센서로부터 수집한 외부 정보와 지상통제센터 AI로부터 받은 정보를 조합해 위협을 식별하고 위험 평가를 수행한 뒤, 최적의 대응 방안을 도출합니다.

# 4. 지상통제센터 AI

지상통제센터 AI는 UAV가 뜨기 전에, 서로 다른 세 곳에서 들어오는 정보를 동시에 모읍니다. GCS(지상통제장비)로부터는 UAV가 실제로 날아갈 경로를, 운용자 단말로부터는 사람이 직접 등록하는 임무 지시서를, C4I(지휘통제통신정보체계)로부터는 현재 적 상황이나 가용 자산 같은 정보를 자동으로 받습니다. 이 세 정보는 서로 다른 시스템에서 오기 때문에 한쪽이 끝나길 기다릴 필요 없이 각자 준비되는 대로 들어옵니다.

이 중 지시서는 사람이 쓴 자유로운 문장이기 때문에, AI가 그 글을 읽고 "이 위협은 이 정도로 심각해 보인다"는 판단 후보를 뽑아냅니다. 다만 이 판단을 곧바로 믿지는 않습니다. C4I로부터 들어온 상황 정보와 서로 맞춰봐서, 같은 내용이 다른 경로로도 확인되면 신뢰도를 더 높이고, 서로 어긋나면 그 사실을 운용자에게 그대로 보여줍니다.

이렇게 GCS·운용자 단말·C4I에서 모이고 다듬어진 정보는 출처별로 정리되어 운용자에게 요약으로 제시됩니다. 운용자는 전체 정보를 한 번 확인하고, AI가 제안한 판단들은 하나하나 근거를 보면서 개별적으로 승인합니다. 이 승인을 거쳐야만 비로소 임무 정보가 확정되어 UAV로 전달됩니다. 정리하면, AI는 판단의 재료를 더 정확하게 다듬어주는 역할까지만 하고, 실제 결정은 항상 사람이 내립니다.

# 5. UAV 레이어

UAV 온보드 AI는 Sensor Layer, Semantic Abstraction Layer, Threat Modeling Layer, Risk Assessment Layer, Response Layer, 총 다섯 개의 레이어로 구성됩니다.

## 1) Sensor Layer

Sensor Layer는 UAV에 탑재된 센서들로부터 원시 데이터를 수집하는 계층입니다. 수집 항목은 여덟 개 종류로 나뉩니다.

**영상 정보(EO/IR)**: 가시광선(EO) 영상(주간 정찰·객체 탐지), 적외선(IR) 영상(야간·열감지·위장목표 탐지), 짐벌 각도(카메라 방위각·앙각)

**항법 정보(Navigation)**: GPS/GNSS+IMU+고도(위경도·고도·속도·자세), 기압계(기압 기준 고도), 지자기/나침반(GPS 끊길 때 방향 유지), 대기속도(풍속 보정용), 웨이포인트 텔레메트리(비행 임무 진행 정보)

**통신 정보(C2 Link)**: 통신 감도/패킷 손실(링크 품질), 통신 Latency(지연시간), 통신 공격 로그(MAVLink DoS·GPS 재밍 트래픽), 통신 주파수(재밍 감지용), 암호화 상태(통신 보안 여부)

**전자전 정보(EW)**: GNSS 스푸핑 지표(confidence 급락·위치 점프), GPS 위성 수신 개수(스푸핑 탐지용), RF 신호 강도(주변 전파환경·재밍 감지)

**드론 상태(Health)**: 배터리 잔량/전압, 온도(배터리·모터 과열), 진동(피격·고장 감지), 모터 RPM(이상 감지), 낙하산/페일세이프 상태(비상대응)

**음향 정보(Acoustic)**: 총기/폭발음(총성·폭발음·헬기·차량 감지), 적 드론 소리(프로펠러 소리 감지)

**환경 정보(Environment)**: 풍속/풍향(비행경로 보정), 기온(배터리 성능 영향), 고도(지면기준, 지형충돌 방지), 지형 고도/경사(DEM, 산악지형 인식·LoS 차단 계산)

**임무 상태(Mission Status)**: 현재 웨이포인트 진행상태, 목표지점까지 남은 거리, 귀환기지까지 남은 거리, 현재 임무단계(이동중/정찰중/귀환중)
