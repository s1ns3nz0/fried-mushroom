# A-5. Endurance RTL Advisory — bingo-fuel 귀환 권고

## 왜 (임팩트)
UAV 안전의 핵심은 **"지금 돌아가지 않으면 못 돌아온다"**(bingo-fuel)를 아는 것이다. 03
`operational_margin` 은 배터리 잔량을 밴딩(sufficient/limited/critical)만 할 뿐, **귀환
가능성**(홈까지 거리 × 남은 에너지)은 계산하지 않는다. 임무 몰입 중 에너지 소진으로 귀환
실패하는 최악의 손실을 막는 **공간적 에너지 추론**이 비어 있었다.

`onboard/endurance.py`가 그 갭을 채운다.

## 무엇
`assess_endurance(raw, mission_brief, *, cruise_speed_mps, reserve_frac, endurance_rated_s)`:
- 현재 위치(`raw.navigation.gps`) → 홈 베이스(`corridor.bases`, home>emergency>alternate) haversine 거리.
- `RTL소요 = 거리 / 순항속도`, `남은비행 = 내구 × 배터리%`, `예비 = 내구 × reserve_frac`.
- `여유(margin) = 남은비행 − 예비 − RTL소요`. **margin ≤ 0 → bingo 도달 → RTL 권고**.

### 예시 (t3 골든)
```
홈 1112m(RTL 65s), 잔여 1404s, 여유 1069s — 임무 지속 가능. (CONTINUE)
멀리+저배터리 → ⚠ bingo 도달 → 지금 RTL 권고. (RTL, rtl_required=True)
```

## CRITICAL — SCC-1 (advisory, 결정 대체 금지)
결정론 RTL 판정(06 Response, 위협 기반)을 **대체하지 않는다**. 에너지 기반 RTL 권고는 병렬
안전지표로 제공하며(운용자/상위 로직이 판단), 입력을 변이하지 않고 `EARTH_RADIUS_M`(읽기
전용)만 참조한다.

## 용도
운용자 bingo 경고, 임무 지속 가능성 게이지, sitrep 융합.

무변이·데이터부재 graceful).
