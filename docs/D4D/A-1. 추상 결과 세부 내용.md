# A-1. 추상 결과 세부 내용

Sensor Abstraction Layer(03)가 한 사이클마다 내보내는 출력의 구조·예시·필드 스펙입니다. 기존에는 채널 11개 메시지마다 `schema_version`/`ts`/`seq`가 매번 반복돼서 데이터가 불필요하게 커졌습니다. 이번 개정에서는 한 사이클에서 공통인 값들을 배열 밖(최상위)으로 빼고, 사이클 자체를 식별하는 `id`를 추가했습니다. 또한 `operational_margin`에 모터 RPM·진동 등 진단용 원값과 낙하산/페일세이프 상태, 기온을 반영했습니다(하단 changelog 참고).

## 공통 envelope (배치 단위, 사이클당 1회)

```json
{
  "schema_version": "1.0",
  "id": "GIREOGI-0704-01-4471",
  "ts": 1730620801200,
  "channels": [
    { "channel": "<채널명>", "state": "normal | degraded | anomaly", "quality": 0.0, "quality_delta": 0.0, "payload": { "...채널별 필드..." } }
  ]
}
```

### 최상위(배치) 필드 스펙

| 필드 | 타입 | 값 | 설명 |
|---|---|---|---|
| `schema_version` | string | `"1.0"` | 스키마 버전 고정값 |
| `id` | string | `"{sortie_id}-{순번}"` | 이 사이클(추상화 결과 묶음)을 식별. 순번은 사이클마다 1씩 증가(예: `GIREOGI-0704-01-4471` → 다음은 `...-4472`). 절대시간이 아니라 상대적 순서만 알면 충분해서 단순화했습니다 |
| `ts` | integer | epoch 밀리초 | 이 사이클이 생성된 시각 |
| `channels` | array | 11개 원소 | 채널별 결과 |

### 채널별(배열 원소) 공통 필드

| 필드 | 타입 | 값 | 설명 |
|---|---|---|---|
| `channel` | enum(string) | `position_consistency`, `link_status`, `rf_spectrum`, `link_integrity`, `encryption_status`, `mission_phase`, `terrain_class`, `proximity_object`, `acoustic_event`, `obstacle_proximity`, `operational_margin` | 물리 센서 1개가 아니라 의미 단위 출력 |
| `state` | enum(string) | `normal`(정상범위) / `degraded`(측정은 되지만 신뢰도 낮음) / `anomaly`(값 자체가 위협·이상 신호) | 채널 공통 3단계 |
| `quality` | float | `0.0`(신뢰불가) ~ `1.0`(완전신뢰) | **판독기/모델 건전성**(센서 자체 신뢰도 또는 AI모델 확신도). **위협 크기와 분리**한다 — 결정론 판독 채널(`encryption_status`, `link_integrity`)은 anomaly여도 판독 신뢰도가 그대로이므로 quality를 낮추지 않고, 이상 증거는 `state`/`payload`로만 전달. 계측기 자체 손상(샘플 부족·링크 두절 등)일 때만 degraded로 quality↓. 위협모델링 채널가중치에 그대로 반영 |
| `quality_delta` ⚠️신규 | float | 전 사이클 대비 `quality` 변화량(`quality(t) - quality(t-1)`, 첫 사이클은 `0.0`) | 새 센서 아님 — quality의 시간차만 계산. 안개(서서한 저하) vs 레이저(순간 급락)를 구분하는 데 사용. Threat Modeling Layer(04)가 T5(레이저) 판정에 사용(-0.3 미만이면 매칭) |
| `payload` | object | 채널마다 다름 | 아래 참조 |

## 전체 출력 예시 (한 사이클, T3+저시정 시나리오)

```json
{
  "schema_version": "1.0",
  "id": "GIREOGI-0704-01-4471",
  "ts": 1730620801200,
  "channels": [
    { "channel": "position_consistency", "state": "normal", "quality": 0.95, "quality_delta": 0.0,
      "payload": { "gps_imu_residual_m": 0.8, "baro_residual_m": 0.3, "airspeed_residual_ms": 0.4, "threshold_m": 5.0, "hdop": 0.9, "vdop": 1.2, "satellite_count": 11, "cn0_avg_db": 42.5 } },

    { "channel": "link_status", "state": "normal", "quality": 0.90, "quality_delta": 0.0,
      "payload": { "rssi_dbm": -62, "noise_floor_dbm": -95, "freq_mhz": 915.0, "packet_loss_rate": 0.01, "latency_ms": 45 } },

    { "channel": "rf_spectrum", "state": "normal", "quality": 0.80, "quality_delta": 0.0,
      "payload": { "wideband_anomaly": false, "bearing_deg": null } },

    { "channel": "link_integrity", "state": "normal", "quality": 0.99, "quality_delta": 0.0,
      "payload": { "checksum_fail_rate": 0.0, "seq_gap_count": 0 } },

    { "channel": "encryption_status", "state": "normal", "quality": 0.99, "quality_delta": 0.0,
      "payload": { "mode": "AES256", "downgrade_detected": false } },

    { "channel": "mission_phase", "state": "normal", "quality": 0.90, "quality_delta": 0.0,
      "payload": { "declared": "LOITER_ROI", "behavioral": "loiter_pattern", "match": true, "mission_phase_confidence": 0.9 } },

    { "channel": "terrain_class", "state": "degraded", "quality": 0.55, "quality_delta": -0.05,
      "payload": { "dominant_class": "open_field", "source": "camera_verified", "gis_last_updated": "2025-11", "camera_mismatch": true, "exposure_score": 0.72, "risk_map_ref": "buf://terrain_seg/4471" } },

    { "channel": "proximity_object", "state": "anomaly", "quality": 0.55, "quality_delta": -0.05,
      "payload": { "class": "person", "weapon_shape": true, "bearing_deg": 142.3, "closing": true, "closure_rate_mps": 3.2, "degraded_reason": "low_visibility" } },

    { "channel": "acoustic_event", "state": "anomaly", "quality": 0.92, "quality_delta": 0.0,
      "payload": { "event_type": "gunshot", "detection_stage": "threshold_only", "peak_db": 118, "bearing_deg": 139.8 } },

    { "channel": "obstacle_proximity", "state": "normal", "quality": 0.85, "quality_delta": 0.0,
      "payload": { "distance_m": null, "closure_rate_mps": null } },

    { "channel": "operational_margin", "state": "degraded", "quality": 1.0, "quality_delta": 0.0,
      "payload": { "battery_pct": 65, "battery_state": "sufficient", "weather_state": "limited", "mechanical_state": "sufficient", "link_margin_state": "sufficient", "time_margin_state": "sufficient", "overall": "limited", "worst_factor": "weather", "failsafe_state": "ready", "diagnostics": { "motor_rpm": [8200, 8150, 8190, 8175], "motor_temp_c": 42.0, "vibration_level": 0.12, "ambient_temp_c": -3.5 } } }
  ]
}
```

이 예시의 스토리: GPS/링크/보안은 정상. terrain_class는 GIS 조회 결과와 카메라 세그멘테이션 결과가 달라(`camera_mismatch:true`) 저시정 속에서 카메라 판정으로 degraded 처리됨. 같은 저시정으로 proximity_object도 quality가 낮지만 사람+무기형태가 감지되어 anomaly. 두 채널의 `quality_delta`가 `-0.05`로 작은 음수인 건 안개가 서서히 짙어지는 상황을 나타냅니다 — 레이저처럼 한 사이클 만에 `-0.3` 이상 급락하는 것과는 패턴이 다릅니다(Threat Modeling Layer 04의 T5 판정 기준). acoustic_event는 임계값매칭만으로 총성이 명확히 확정(`detection_stage:"threshold_only"`)되어 YAMNet까지 갈 필요가 없었던 경우. operational_margin은 기온이 낮아(`ambient_temp_c:-3.5`) weather_state가 limited로 내려간 경우 — 모터·진동은 정상이라 mechanical_state는 sufficient, 페일세이프도 정상 대기 상태.

다음 사이클의 `id`는 `GIREOGI-0704-01-4472`처럼 마지막 숫자만 1씩 늘어납니다.

## 채널별 payload 필드 스펙

### position_consistency — 🔵 결정론적

GPS 위치를 IMU 관성항법 추정치·기압고도와 서로 대조해서 GPS 값 자체의 스푸핑 여부를 판정합니다.

| 필드 | 타입 | 값 | 설명 |
|---|---|---|---|
| `gps_imu_residual_m` | float | ≥0, 미터 | GPS 위치와 IMU 관성항법 추정 위치의 차이. `threshold_m` 초과 시 스푸핑 의심 |
| `baro_residual_m` | float | ≥0, 미터 | GPS 고도와 기압고도계 추정고도의 차이 |
| `airspeed_residual_ms` | float | ≥0, m/s | GPS/IMU 속도추정과 대기속도 추정치의 차이 |
| `threshold_m` | float | 고정설정값(기본 5.0) | 위 세 잔차의 판정 임계값. 초과 시 `state=anomaly` |
| `hdop` | float | ≥0(통상 2 미만 양호) | 수평 정밀도저하율 |
| `vdop` | float | ≥0 | 수직 정밀도저하율 |
| `satellite_count` | integer | ≥0(통상 6개 이상 안정적) | 포착 위성 수 |
| `cn0_avg_db` | float | 통상 35dB-Hz 이상 양호 | 위성 신호대잡음비 평균. 판정의 보조 컨텍스트로만 사용, 자체 트리거 아님 |

### link_status — 🔵 결정론적

| 필드 | 타입 | 값 | 설명 |
|---|---|---|---|
| `rssi_dbm` | float | 통상 -120(매우약함)~-30(매우강함) | 수신신호강도 |
| `noise_floor_dbm` | float | 통상 -110~-80 | 배경잡음 레벨 |
| `freq_mhz` | float | 예: 915.0, 2400.0 | 현재 사용 주파수(홉핑 상태 반영) |
| `packet_loss_rate` | float | 0.0~1.0 | 통상 0.1 이상이면 degraded 트리거 |
| `latency_ms` | float | ≥0, 밀리초 | 값이 클수록 링크 열화 |

### rf_spectrum — 🔵 결정론적

| 필드 | 타입 | 값 | 설명 |
|---|---|---|---|
| `wideband_anomaly` | bool | `true`(카탈로그에 없는 신규 RF패턴 — 재밍·미상신호원 의심) / `false`(정상) | |
| `bearing_deg` | float \| null | 0~360, 방향성 안테나 없으면 `null` | 이상신호원 방위각 |

### link_integrity — 🔵 결정론적

| 필드 | 타입 | 값 | 설명 |
|---|---|---|---|
| `checksum_fail_rate` | float | 0.0~1.0 | 체크섬/서명 검증 실패 비율. 급증 시 변조·재밍 의심 |
| `seq_gap_count` | integer | ≥0 | 시퀀스번호 누락 건수 |

### encryption_status — 🔵 결정론적

| 필드 | 타입 | 값 | 설명 |
|---|---|---|---|
| `mode` | enum(string) | `AES256`(정상 운용모드) / `AES128`(구모드, 정책상 허용여부 확인 필요) / `NONE`(암호화 해제 — 즉시 anomaly) | |
| `downgrade_detected` | bool | `true`(강제 다운그레이드 이력 — 공격 의심) / `false` | |

### mission_phase — 🔵 결정론적 룰테이블

| 필드 | 타입 | 값 | 설명 |
|---|---|---|---|
| `declared` | enum(string) | `TAKEOFF`, `WAYPOINT`, `LOITER_ROI`, `RTL`, `LAND` | 오토파일럿이 보고하는 현재 임무단계 |
| `behavioral` | enum(string) | `cruise_pattern`(이동중) / `loiter_pattern`(선회·체공중) / `unknown`(불명확) | 헤딩·속도·짐벌상태에서 역산한 실제 행동양상 |
| `match` | bool | `true`(일치) / `false`(불일치 — 오토파일럿 이상 또는 하이재킹 의심) | |
| `mission_phase_confidence` | float | `0.9`(완전일치) / `0.6`(부분일치) / `0.3`(명백한불일치), 이산값 3종 | |

목표지점·귀환기지까지 남은 거리는 이 채널에 없습니다. 현재 위치(position_consistency)와 항로정보(mission_brief의 corridor·bases)를 합쳐 계산해야 하는 값이라, Sensor Abstraction Layer 범위를 벗어납니다 — Threat Modeling Layer(04)나 위험평가 단계에서 다룰 예정입니다.

### terrain_class — 🔵 GIS 우선 + 🟡 카메라 세그멘테이션(검증·보조)

| 필드 | 타입 | 값 | 설명 |
|---|---|---|---|
| `dominant_class` | enum(string) | `open_field`(개활지, 노출도 높음) / `forest`(삼림, 은폐 양호) / `urban`(시가지, 근접위협 밀도 높음) / `water`(수상, 착륙지 아님) / `obstacle`(장애물 밀집) | |
| `source` | enum(string) | `gis_lookup`(사전 지형DB 조회, 기본값) / `camera_verified`(카메라 세그멘테이션 검증·보정, GIS와 불일치 시에만 발생) | |
| `gis_last_updated` | string | 예: `"2025-11"` | 참조 GIS 데이터 조사·갱신 시점 |
| `camera_mismatch` | bool | `true`(카메라 결과가 GIS와 다름 — `state=degraded`로 사람 확인 유도) / `false` | |
| `exposure_score` | float | 0.0~1.0 | 은폐 없이 노출되는 정도 |
| `risk_map_ref` | string \| null | 버퍼/공유메모리 참조 문자열 | 카메라 세그멘테이션 실행 시에만 채워짐, GIS-only면 `null` |

### proximity_object — 🟡 AI(YOLOv8n/YOLO-NAS nano → 사람 bbox 크롭 → MobileNetV3-small 무기판별)

| 필드 | 타입 | 값 | 설명 |
|---|---|---|---|
| `class` | enum(string) | `none`(감지없음) / `person` / `drone`(소형무인기) / `vehicle` / `projectile`(발사체) | |
| `weapon_shape` | bool | `class=person`일 때만 의미. `true`(무기형태 소지) / `false` | |
| `bearing_deg` | float | 0~360 | 기체 기준 대상 방위각 |
| `closing` | bool | `true`(접근중) / `false` | |
| `closure_rate_mps` | float | 양수=접근속도, 음수=이탈속도 | |
| `degraded_reason` | string \| null | 예: `"low_visibility"`, 정상이면 `null` | |

### acoustic_event — 🔵 1차 임계값매칭 + 🟡 YAMNet(애매한 경우만 2차 확인)

| 필드 | 타입 | 값 | 설명 |
|---|---|---|---|
| `event_type` | enum(string) | `none`(이벤트없음) / `gunshot`(총성) / `propeller_approach`(타 기체 접근 프로펠러음) | |
| `detection_stage` | enum(string) | `threshold_only`(1차 임계값매칭만으로 확정) / `ai_confirmed`(1차매칭이 애매해 YAMNet 2차확인까지 거침) | |
| `peak_db` | float | dB SPL | 피크 음압 |
| `bearing_deg` | float \| null | 0~360, 어레이마이크 없으면 `null` | |

### obstacle_proximity — 🔵 결정론적(레인지파인더) 또는 🟡 AI(단안 깊이추정 사용 시)

| 필드 | 타입 | 값 | 설명 |
|---|---|---|---|
| `distance_m` | float \| null | ≥0, 장애물 없으면 `null` | |
| `closure_rate_mps` | float \| null | 장애물 없으면 `null` | 양수=접근, 음수=이탈 |

### operational_margin (집계, 위협채널 아님) — 🔵 결정론적 worst-case 집계

| 필드 | 타입 | 값 | 설명 |
|---|---|---|---|
| `battery_pct` | float | 0~100 | |
| `battery_state` | enum(string) | `sufficient`(충분) / `limited`(제한, 주의) / `critical`(위험, 즉시판단) | |
| `weather_state` | enum(string) | `sufficient` / `limited` / `critical` | 시정·풍속·기온 기반(원값은 `diagnostics.ambient_temp_c` 참조) |
| `mechanical_state` | enum(string) | `sufficient` / `limited` / `critical` | 모터·진동 기반(원값은 `diagnostics` 참조) |
| `link_margin_state` | enum(string) | `sufficient` / `limited` / `critical` | 통신마진 기반 |
| `time_margin_state` | enum(string) | `sufficient` / `limited` / `critical` | 임무경과·일몰시각 기반 |
| `overall` | enum(string) | `sufficient` / `limited` / `critical` | 위 5개 중 worst-case |
| `worst_factor` | enum(string) | `battery`, `weather`, `mechanical`, `link`, `time` | 어떤 항목이 worst였는지 |
| `failsafe_state` ⚠️신규 | enum(string) | `ready`(정상 대기) / `triggered`(작동됨) / `fault`(고장) | 낙하산·페일세이프 상태. 이산적 안전상태라 worst-case 집계엔 넣지 않고 항상 별도로 노출 |
| `diagnostics` ⚠️신규 | object | `{motor_rpm: float[], motor_temp_c: float, vibration_level: float, ambient_temp_c: float}` | `mechanical_state`/`weather_state` 계산에 쓰인 원값. 판단 트리거가 아니라 사람이 "왜 이 상태가 나왔는지" 확인하는 진단용. `motor_rpm`은 모터별 배열 |

## 참고 — 이 출력을 소비하는 쪽 (위협모델링 Step D 출력 스키마)

이 채널 결과는 위협모델링 Step B~D를 거쳐 아래 형태로 압축됩니다. `04. Threat Modeling`에서 candidates·primary·background_exposure_score로 구성된 최신 스키마로 갱신됐습니다 — 상세는 04 문서 참고.

```json
{
  "declared_phase": "LOITER_ROI",
  "mission_phase_confidence": 0.9,
  "candidates": [
    { "threat_event": "T3", "match_count": 2, "confidence": 0.917, "confidence_source": "ai",
      "kill_chain_stage": "후기", "potential_outcome": "attrition_kill" }
```
